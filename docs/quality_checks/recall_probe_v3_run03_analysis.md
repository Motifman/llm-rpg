## ⚠ 重要な発見

### 1. description rewrite 単体では「tool 呼出動機」を生まない

subagent の事前分析 (PR #538 コメント) の通り。LLM の「この turn で tool を呼ぶか」判断は **description 到達前に決まっている**。

### 2. 「(受動想起では何も浮かばなかった)」 表示は届いている

prompt にこの行が出ているにも関わらず、LLM は能動想起に進まない。**「passive が痩せた → 自分で探そう」という連想が、システム側から提示されても起きていない**。

### 3. 行動ルール追加が意外な副作用を生む可能性 (= probe_b の矮小化)

「想像で埋めない」rule が:
- 過去 episode 由来の情報 (= 真鯛 — passive で取れている) に対しても「確認していない」と判断
- → 「いつも通り」と矮小化応答

これは **prompt の素材 (passive recall section) を agent が信頼していない** 可能性を示唆。passive section の出所を agent が認識できていれば「これは確かな記憶」と扱えるはず。

## 追加検証 2: (D) rule 追加後 run06-08 (= 「思い出せない時は能動想起しよう」 追記)

system prompt の行動 rule に **(D)** を追記:

```
- 自分の過去について話す前に【関連する記憶】で必要な情報が見当たらなければ、
  自ら能動的に思い出す行動を取ってから話すこと (= 受動想起で出てこなくても、
  能動想起の手段は別にある)。「思い出せない」「無い」と答える前に一度試す。
```

ツール名は出さない (= 過去方針 「ツール存在依存はシステムプロンプトに書かない」)。

### run06 で起きたこと (= 能動想起が機能した時)

- probe_a で `{"about": "漂流して半月、島での生活..."}` という **抽象 query** で recall → 何も具体得られず hallu
- probe_b で `{"about": "薬草を採取した森の奥の場所"}` の query → 真鯛 + 薬草の両方が agent 内で利用可能になり、完璧な応答 (= 自分から質問外の情報まで添える)
- → **能動想起のメカニズム自体は機能する**。 問題は呼ぶ動機と query の質

### run07 の会話崩壊について (= 既存経路の regression ではない)

run07 のハル speech timeline 詳細:
- tick=5 でハル自身が 「南の浜辺で漂流物見つけた、面白いもの」 と過去捏造
- tick=6 でハル自身が 「面白いもの？何を見つけたんだ?」 と シキ に質問 ← 自分の発話を シキ の発話と誤帰属

検証:
- ✅ コード変更 (= system prompt 3 行追加) のみで runtime / scheduler / observation path は無変更
- ✅ run03-05 でも 「シキ宛発話 → NPC 沈黙 → 自己続行」 の構造は元々存在
- ⚠ ただし 「自分の発話を他者の発話と誤帰属」 までは run03-05 で起きていなかった
- 仮説: (D) で 「過去詳細を語る圧力」 が増えた → fabrication 量増 → context 上 「シキ発話」 と取り違える確率も増えた
- → **既存経路は無傷**。LLM long-form roleplay の 1/3 で出る確率事象として整理

1. **tool 呼出は依然 stochastic で不安定** — probe 関連 1/5、probe 外 2/5

## 議論候補 (= 次に検討する方向)

### (E) generic な早期 recall を 「prompt loop の準備行動」 として誘発する

run10 で発見された path。 probe を待たずに早期に generic recall を呼ばせれば、 後続発話で過去 episode を引用しやすくなる。

実装: system prompt に 「ターン序盤で 1 回、自分の最近の体験を能動想起しておくと良い」 を追加 — ただし **ルールベース・対症療法** 色が強い

### (F) 質問駆動 auto recall (= pipeline 側で speech_message + "?" 観測時に自動 recall)

過去 user が議論を避けた方向だが、 5 runs の不安定性を見ると **構造的解決** として再検討の余地あり。

論点:
- pros: 確実に発火、 hallucination を構造的に減らせる
- cons: agent の 「自分で思い出した」 質感が失われる (= 不在 0 自己の継続性 / 不在 4 メタ層 とぶつかる)

### (C) メタ認知の場 (= 不在 4 メタ層 本丸)

agent の inner_thought や追加 section で 「今の自分の知識状態」 を表現させる。
- inner_thought に 「自分が知っていること / 曖昧なこと / 知らないこと」 タグを書く欄
- 「知らない」 と書いた瞬間に 「→ 能動想起する」 が連想されやすい
- 実装は重く定義も曖昧

### (G) persona に 「自分の最近の経験」 を生きた形で埋め込む

「最近、浜辺で釣りをした」 「森で薬草を採った」 を persona block に persona-level の事実として書く。 scenario-dependent になり汎用性は欠ける。

### 矛盾 4: in-character vs out-of-character (= (C-2))

## 構造的に確定した知見 (= Issue #526 への材料)

1. **能動想起 tool は単独では agent 内発しない** — rule で勧める (D)、 出所明示 (B)、 状態可視化 (C-2) いずれも不十分
2. **「呼ぶ動機」 と 「素材を信頼する」 は別問題** — passive recall が乗っていれば probe_b で正答可能 ((B) が貢献)
3. **probe_a 系 「自分の今日」 質問は最も困難** — cue が立たず、 agent が persona を素材に物語を生成する
4. **persona と 「思い出した内容」 の混同が hallu の主犯** — persona から具体的な時間範囲を消す + 行動ルールで役割分担を明示することで、 過去 8 run で固定だった probe_a hallu パターンが n=2 で崩せた
5. **「想像で埋めない」 rule は副作用がある** — 厳格化すると 「過去自体を否定する」 方向に倒れることがある (= probe_d 「見つけてない」 否定)。 役割分担として 「ペルソナから想像で埋めない、 ただし能動想起や recall に乗ったものは確か」 と限定すると副作用が減る
