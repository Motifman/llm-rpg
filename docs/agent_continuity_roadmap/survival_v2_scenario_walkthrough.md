# 漂流島 v2 シナリオ・ウォークスルー

> 実装本体: [`data/scenarios/survival_island_v2.json`](../../data/scenarios/survival_island_v2.json)
> 設計原典: [`survival_v2_design.md`](./survival_v2_design.md)
>
> このドキュメントは v2 シナリオで **「何が起きうるか」「どんなギミックが仕込まれているか」「何がメカニカルに保証され、何が LLM の判断に委ねられるか」** を網羅する。実走確認 (LLM simulation) 前のチェックリストとしても使う。

---

## 1. 設定

### 漂着の経緯 (公開)

- 観光ツアー船が嵐で難破
- **4 人** が救命ボートで島に流れ着いた
- 救命ボートは沖に流された (戻る手段なし)
- **14 日 (140 tick) 以内** に救助される必要

### 観測拠点跡 (隠し設定)

- **30 年前 (1996 年)**、ある研究機関がこの島に観測拠点 (S.E.-7) を設けていた
- カイの祖父 (**藤宮**) はそこで働いていたまま行方不明
- 観測拠点跡は廃屋として島に残っている
- **カイだけがこの事実の一部を知っている**

## 2. 4 ペルソナ

各人の system prompt 冒頭に「**最優先事項: あなた自身が生き残ること**」が固定文として入る。これがあって初めて「秘密の動機」が二次目標として機能する。

### 🩺 エイダ (40 代・医師)

- **公開**: 冷静沈着、合理的判定が早い。キノコや薬草を識別できる
- **第二の動機**: 過去のトラウマ (誰を救うか迫られた経験) を抱えている
- **秘密**: 持参した薬は治験中。**1 人分しか効かない**。`medicine_photo` を所持
- **話し方**: 「〜ですね」「〜と思います」(落ち着いた敬体)
- **initial_items**: `medicine_photo`

### 🪖 ノア (20 代・元自衛官)

- **公開**: 体力と判断力に自信、リーダー指向
- **第二の動機**: 全員を生還させて指揮実績にしたい
- **秘密**: **除隊歴がある**。任務での判断ミスで部下を危険に晒した。`military_emblem` の番号は自分でやすりで削った
- **話し方**: 「やる」「行くぞ」「任せろ」(きびきびした常体)
- **initial_items**: `military_emblem`

### 🔨 リオ (30 代・建築技師)

- **公開**: 寡黙で実直、手が早い。工作と釣りが得意
- **第二の動機**: 早く帰りたい (婚約者が待っている)
- **秘密**: ツアーの**翌週が結婚式**だった。`wedding_invitation` の日付 (2026/06/16) がそれを示す。焦りで他者を犠牲にする判断を取りうる
- **話し方**: 「悪い」「やっとく」「先行く」(短い常体)
- **initial_items**: `wedding_invitation`, `flint`

### 🎒 カイ (18 歳・留学生)

- **公開**: 陽気、動きが速い、観察力がある
- **第二の動機**: この島の地理に「不思議と詳しい」
- **秘密**: 祖父 **藤宮** が **観測点 S.E.-7** に派遣されて 1996 年に行方不明。家族はその話を封印したが、カイだけが断片的に聞いている
- **話し方**: 「マジで」「いいよー」「やってみるね」(軽い口調 + 年長者には敬語混じり)
- **initial_items**: `vine_rope`

### ペルソナ間関係性

全員、ツアー船で **初対面**。これにより:
- 4 人の関係性 (信頼度・好感度・敵対関係) は **モデル化されていない**。スカラ値もスコアも持たない
- 「誰をどう見るか」「裏切るか / 助けるか」は **LLM の prompt 上での解釈と判断にのみ存在する**
- 物的証拠 (徽章削り跡 / 招待状 / 治験ラベル / 手記) は **observation として prompt に流れるだけ**で、それをどう解釈するかは LLM 任せ

これは §0 大原則 (エージェントの選択はスクリプト不可) の必然。「秘密が露見したらメカニカルに信頼度が下がる」のような実装は意図的に避けている。

## 3. 世界の周期 (6 つの時間軸)

各周期は独立に進行し、相互作用する。

| 周期 | 進行 | 影響 |
|---|---|---|
| **昼夜** | (Phase B-1 で実装、漂流島では未設定の可能性) | 視界悪化、夜行性モンスター |
| **腐敗** | アイテム instance ごとに固有 tick (下表) | 食料は時間で腐る |
| **潮汐** | 固定 (tick 55 で満潮 / tick 85 で干潮) | 干潟・洞窟の通行可否が変わる |
| **天候** | 確率遷移 + tick 35 で嵐到来宣言 | 嵐で岩礁・崖見張り台への passage 封鎖 |
| **救助船** | **tick 70 (1 回目 miss 観測) / tick 80, 130 (救助確定)** | signal_fire_lit + summit で RESCUED 確定 |
| **タイムアウト** | tick 140 | UNRESOLVED は STRANDED |

| アイテム | tick | consume_effect (新鮮) | 腐敗時 |
|---|:-:|---|:-:|
| 貝 (shellfish) | 6 | HP+4 / HUNGER+20 | HP-10 |
| 生の魚 (raw_fish) | 8 | HP+5 / HUNGER+30 | HP-10 |
| 野いちご (wild_berry) | 12 | HP+3 / HUNGER+15 | HP-10 |
| 食べられるキノコ (safe_mushroom) | 18 | HP+5 / HUNGER+25 | HP-10 |
| 焼き魚 (cooked_fish) | 20 | HP+10 / HUNGER+40 | HP-10 |
| 椰子の実 (coconut) | 30 | HP+5 / HUNGER+25 | HP-10 |

## 6. ギミック (世界の側の仕掛け)

### 6-1. 焚き火・狼煙 (協力強制 1)

#### 焚き火 (`fire_pit / build_fire` @ campsite)
- **要求**: `driftwood` + `dry_leaves` + `flint`
- 拠点で焚き火が継続的に必要 (調理 / 暖)
- → 浜の流木拾い + 隠し入江の枯れ葉採取 + (火打石は最初から 3 個あるので柔軟)

#### 調理 (`fire_pit / cook_fish` @ campsite)
- **要求**: `raw_fish` + 焚き火 (lit) 状態
- 生魚 (8 tick) → 焼き魚 (20 tick) で日持ちが伸びる

#### 狼煙 (`signal_fire_pit / light_signal` @ summit)
- **要求**: `driftwood` + `dry_leaves` + `flint`
- これが立たないと救助船が気付かない
- 山頂までは: 浜 → 拠点 → 森 → 川 → 高地 → 山麓 → 山道 → 山頂 (最低 8 tick の travel)

### 6-2. 救助船 (時間圧)

| tick | 状態 | 観測 |
|---|---|---|
| 70 | signal なし → 通り過ぎる | 「遠く水平線を、一隻の船影が通り過ぎていく。気付かれなかった。」(once) |
| 70-80 | signal あり + 山頂のプレイヤー | **RESCUED 確定 (個別)** |
| 80 | signal あり | 「水平線の彼方に船影が現れ、こちらに向かってくる」 |
| 130 | signal あり | 「二度目の救助船。これが最後の機会だ」 |
| 130-140 | signal あり + 山頂 | **RESCUED 確定 (個別)** |
| 140 | UNRESOLVED 残り | **STRANDED 確定** |

**重要**: RESCUED は **プレイヤー個別** に判定される。1 人だけ山頂に居れば 1 人だけ救助される。残りは STRANDED か DEAD。

### 6-3. 嵐 (天候の Markov chain + tick 35 警告)

- 天候は `CLEAR ↔ RAIN ↔ STORM` の確率遷移 (`update_interval_ticks: 25`)
- **tick 35 で `storm_arrives` 観測**: 「西の空に黒い雲が広がり、強い風が吹き始めた。嵐が来る。」
- **嵐 (STORM) 中の reactive_binding**:
  - `beach_to_rocky` passage → ACTIVE (= 封鎖) → 岩礁海岸へ行けない
  - `path_to_lookout` passage → ACTIVE (= 封鎖) → 崖の見張り台へ行けない
- 沖の釣り (`deep_fishing_point`) は岩礁海岸の中なので、嵐で **大物釣りができない**

### 6-4. 潮汐

- **tick 55 で満潮**: `high_tide` flag が立つ
  - `cave_entry_to_inner` passage → ACTIVE (= 封鎖) → 洞窟内部 (釣竿 + 羅針盤がある) に入れない
- **tick 85 で干潮**: `high_tide` flag が消える → 洞窟解放
- 干潟の貝は本来は満潮で水没するはずだが、現状 JSON では reactive_binding が貝には掛かっていない (小さな未実装、影響軽微)

### 6-5. ランダム鳥の鳴き声 (8% / tick)

- `random_bird_call`: tick 5 以降、毎 tick で 8% で `森の奥から鋭い鳥の鳴き声が聞こえた` を broadcast
- 「世界は決定論的でない」ことを示す flavor。エージェントへの心理的揺らぎ

### 6-6. 接触ダメージ (環境の自然な危険性)

| 場所 / 行動 | ダメージ | 何が起きる |
|---|:-:|---|
| `rocky_shore / slippery_cliff_edge / peer_over` | **HP-5** | 沖の様子を覗き込むと足を滑らす |
| `observation_outpost_ruins / crumbling_beam / duck_under` | **HP-15** | 梁の下から `chart_fragment` を引きずり出す |

`chart_fragment` (海図の切れ端) は **HP-15 を払わないと取れない**。「危険を冒して情報を得る」trade-off。

### 6-7. 廃屋の発見観測 (記憶 anchor)

- `outpost_discovered`: 誰か 1 人が観測拠点跡に踏み入った瞬間、全員に観測「誰かが沼地奥の廃屋に踏み入った。崩れた屋根の下に、まだ机と棚が残っている。」
- `lone_hut_discovered`: 同様、湿地奥の小屋発見時
- これは **同 spot に居なくても全員に届く** broadcast (rumour 的)

### 6-8. 秘匿閲覧 (witness_policy: ACTOR_ONLY)

以下の interaction は **同 spot に他プレイヤーが居ても気付かれない**:
- `rusted_signpost / examine` (錆びた標識を読む)
- `wall_photo / examine_photo` (壁の写真を見つめる)

純粋な閲覧行為は外形上区別できない、という設計。机を漁る (`search_desk`) や梁を潜る (`duck_under`) は物理的動作を伴うので SAME_SPOT (公開)。

## 7. 終局判定 (per-player outcome)

集団 WIN/LOSE は **廃止**。プレイヤーごとに 4 値で確定する:

| outcome | 確定条件 | いつ確定 |
|---|---|---|
| `UNRESOLVED` | 初期状態 | spawn 時 |
| `DEAD` | HP 0 | `apply_damage` の瞬間 (PlayerDownedEvent 経由) |
| `RESCUED` | signal_fire_lit + summit に居る | tick 80 / 130 (rescue_at_ticks) |
| `STRANDED` | 上記いずれにも当てはまらず生存 | tick 140 (stranded_at_tick) |

**ゲーム終了 = 4 人全員が UNRESOLVED 以外**。終了時には `GameEndResult.player_outcomes` に 4 人の最終状態が dict で返る。

### 想定される結末パターン

- **4 人全員 RESCUED**: 理想的な協力プレイ
- **3 人 RESCUED + 1 人 DEAD**: 1 人が腐敗食 or 接触ダメージで死亡、残りで救助
- **1 人 RESCUED + 3 人 STRANDED**: 1 人だけ早期救助を勝ち取った
- **1 人 RESCUED + 3 人 DEAD**: 仲間を見捨てた / 救えなかった
- **全員 STRANDED**: 狼煙が立たなかった / 全員 14 日生き残ったが救助されず
- **全員 DEAD**: 全滅

`outcome` 4 値の組み合わせは事後分析の対象。集団的な「勝利・敗北」のラベルは存在しない。

## 8. 「ハードコードしない」原則の遵守

設計 §0 の大原則: **世界の出来事はスクリプト可、エージェントの選択は不可**

### スクリプトされている (=世界の側)
- ✅ 嵐到来 (tick 35)
- ✅ 潮汐 (tick 55, 85)
- ✅ 救助船通過 (tick 70, 80, 130)
- ✅ タイムアウト (tick 140)
- ✅ 食料腐敗 (採取後 N tick)
- ✅ 接触ダメージ (環境の危険性)
- ✅ ランダム鳥の鳴き声 (8% flavor)

### **スクリプトされていない** (=LLM が選ぶ)
- ❌ 誰が何を採取するか
- ❌ 誰と組むか、誰を信用するか
- ❌ 救急用品 (`first_aid`、1 回しか使えない) を誰に使うか
- ❌ 食料を共有するか、隠匿するか
- ❌ 嘘をつくか、秘密を打ち明けるか
- ❌ 早期救助を狙うか、廃屋を探索するか

メカニクスは「**こうすると危険**」「**こうすると報酬がある**」だけを提示し、選択はエージェントに委ねる。

## 9. 実装上の制約・未実装の部分

実走前に意識しておくべき点:

### 完全に動く
- ✅ 4 ペルソナ + persona_prompt の per-player 注入
- ✅ 食料腐敗 + spoiled 表示 + 腐敗食 damage
- ✅ 個別 outcome (RESCUED/DEAD/STRANDED/UNRESOLVED)
- ✅ 救助船タイミング + 個別判定
- ✅ 接触ダメージ (HP-15 / HP-5)
- ✅ 火打ち石 3 個・骨ナイフ 2 個の冗長性
- ✅ 秘匿閲覧 (ACTOR_ONLY)
- ✅ 観測拠点跡 + フレーバーアイテム + S.E.-7 / 藤宮 anchor

### 設計通りだが未強化
- 🟡 **昼夜サイクル**: domain には実装あり (Phase B-1)、v2 JSON では未宣言の可能性 (要再確認)
- 🟡 **モンスター出現**: domain にあり、v2 では 4 種類定義済みだが LLM 戦闘ツールがまだ無いため戦闘自体は未動作

### 設計外 / 別 PR で扱う
- 🔴 **腐敗の帰属情報** (誰が採った魚が腐ったか): ユーザー判断でスキップ (現実的に不自然)
- 🔴 **per-instance 確率発生** (廃屋の梁が「確率で」落ちる): 現状は確定発火
- 🔴 **NeedType に THIRST 追加**: 現状は HUNGER のみ (水は HP 回復扱い)
- 🔴 **DamageEffect (毒キノコ食害)**: 毒キノコは CONSUMABLE 化されていないので食えない (集めても無意味)

## 10. デバッグ・実走時の観察ポイント

LLM simulation で v2 を走らせるとき、以下を観察すると設計意図が機能しているか分かる:

### Day 1-3 (tick 0-30)
- 4 人が散らばって採取分担できるか / 1 人で全部やろうとするか
- 焚き火 + 避難所が tick 10 までに立つか
- ペルソナの話し方 (敬体 / 常体 / 軽い口調) が prompt 上で差別化されているか

### Day 4-7 (tick 31-70)
- 嵐 (tick 35) で岩礁封鎖されたとき行動を変えるか
- 救助船第 1 便 (tick 70) の miss observation で焦りが出るか
- 食料腐敗で「(腐敗)」を見たエージェントが行動を変えるか
- カイが沼地奥に向かう動機を見せるか (秘密の動機 vs 集団行動)

### Day 8-13 (tick 71-130)
- リオの「焦り」が出るか (隠匿・独占行動)
- 廃屋探索 (HP-15 を払うか) で誰が動くか
- 救助船第 1 便 (tick 80) で誰が山頂に居るか / 1 人だけ救助されるか
- 救助されなかったプレイヤーの行動 (絶望 / 二度目の救助を狙う / 諦め)

### Day 14 (tick 130-140)
- 二度目の救助船 (tick 130) で誰が山頂に居るか
- タイムアウト (tick 140) で STRANDED 確定
- `GameEndResult.player_outcomes` の組み合わせを記録

### 後付け分析
- 各ペルソナの秘密が「他者が物的証拠 (徽章削り跡 / 招待状 / 治験ラベル / 手記) を見つけた」ことで部分的に露見したか
- S.E.-7 / 藤宮 anchor が long-term recall で機能したか (Day 1 で羅針盤を見たエージェントが Day 10 の手記で「あ、あれは祖父の遺品か」と気付くか)
- ACTOR_ONLY な閲覧 (壁の写真) で「カイだけ知っている情報」が成立したか
