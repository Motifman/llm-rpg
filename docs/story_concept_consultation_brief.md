# ストーリーコンセプト — 外部相談用ブリーフ

> **目的**: 外部の AI サービス（ChatGPT, Claude 等）と共にストーリーをブラッシュアップするための事前情報資料。
> **最終更新**: 2026-04-05

---

## 1. プロジェクト概要

AI キャラクターが仮想世界で自律的に生活・行動するシミュレーションゲーム。プレイヤー（人間）は観察者・干渉者として関わる。

### ゲームの構造

```
┌────────────────────────────────┐
│          現実世界               │
│    （人間プレイヤーの側）        │
│                                │
│    ┌──── 何らかの手段 ────┐     │
│    │  ？？？               │     │
│    ▼                      │     │
│  ┌─────────────────────┐  │     │
│  │   仮想世界            │  │     │
│  │                      │  │     │
│  │  ┌───────────┐       │  │     │

│  │  │ (ロビー)    │        │     │
│  │  │ AIキャラが  │        │     │
```

## 2. 確定している設定

### 荘園（ロビーワールド）

- AI キャラクターたちが共同生活を送る閉じた空間
- **雰囲気**: 温かみのあるファンタジー風の屋敷 + 底に流れる不穏さ + 神秘的な要素
- **世界観キーワード**: 霧、古い洋館、魔法の痕跡、閉ざされた庭園、夜が長い
- 外の世界に出ることはできない（閉じ込められている）
- 時間はゲーム起動中のみ進行する

### AI キャラクターの状態

- **完全な記憶喪失**: ここに来る前のことは一切覚えていない
- 閉じ込められていることは**自覚している**
- 他の住人との関係は時間と共に深まる
- 脱出ワールドへは「気を失い、目覚めたら別の場所にいた」という体験で転送される
- 脱出ワールドでの経験は記憶として保持される

### 人間プレイヤーの立場

- **管理者ではない**
- **異世界からコンタクトを取ってくる存在**
- AI キャラクターからは「天の声」「夢の向こうの人」のように認識される
- 直接世界に触れることはできないが、コミュニケーションは取れる
- 敵意のある存在ではないと認識されている

## 3. ブラッシュアップが必要な部分

### 3.1 人間と仮想世界の接続経緯（「なんやかんや」の部分）

**採用した方向性: 案C — 夢の延長**

現在の案:
> 繰り返し見る不思議な夢——霧の中の荘園。ある夜、夢の中で端末（このゲーム自体）を見つけて起動する。それ以降は現実でもアクセスできるようになった。プレイヤー自身も「夢と現実の境界にいる存在」として曖昧に位置づけられる。ゲームを閉じることは「目を覚ますこと」に対応し、荘園の時間が止まることとも整合する。

**深掘りしたいポイント:**
- 「繰り返し見る夢」の具体的な描写は？ 何がきっかけで夢を見始めた？
- 夢の中で端末を見つけるシーンの具体的な描写
- なぜこのプレイヤーだけが接続できたのか？ 偶然？ 必然？
- 荘園の住人はプレイヤーの存在をどのように初めて知る？
- プレイヤーと住人の初めてのコンタクトはどんな体験？
- 「夢と現実の境界にいる存在」をゲーム内でどう表現する？
- この接続には代償や制約があるか？

### 3.2 荘園の背景設定

**深掘りしたいポイント:**
- この荘園はいつからある？ 誰が作った？ なぜキャラクターがここにいる？
- 不穏さの正体は何？ 具体的にどんな異変や違和感がある？
- 外の世界（荘園の外）はどうなっている？ 霧の向こうに何がある？
- 荘園自体に意思や秘密はある？
- 「脱出ワールドへの転送」は誰がどんな手段で行っている？ 魔法？ テクノロジー？
- 転送の目的は？ 実験？ 試練？ 何かの条件を満たすため？

### 3.3 ゲームタイトル

**決定の手がかりになる要素:**
- 人間が「夢を通じてアクセスする」というメカニズム
- AI キャラクターは記憶喪失で閉じ込められている
- 温かいファンタジー + 不穏さ + 神秘性
- 観察と干渉のゲーム体験
- 「仮想世界のキャラクターが本当に生きているように見える」という最終目標

**タイトル案のためのキーワード:**
- 夢、霧、荘園、声、境界、記憶、鏡、窓、目覚め、眠り
- 英語: Dream, Mist, Manor, Voice, Threshold, Memory, Mirror, Window, Awakening, Slumber

## 4. トーンと表現のガイドライン

### 全体のトーン

- **幻想的だが地に足のついた語り口**。過度にポエティックにならない
- **不穏さは匂わせる程度**。直接的なホラー描写は避ける
- **温かさと寂しさの共存**。キャラクターたちは閉じ込められているが、日々の暮らしには小さな喜びがある
- **謎は急いで解かない**。プレイヤーが自然に疑問を持つように情報を小出しにする

### 避けるべき表現

- 「AI」「プログラム」「シミュレーション」— 世界観を壊すメタ的な言及
- 過度に暗い/残酷な描写 — 不穏さは雰囲気で表現する
- 説明的すぎるナレーション — プレイヤーに推察の余地を残す

### 参考になる作品の雰囲気（方向性の参考として）

- 『約束のネバーランド』の序盤の日常パート — 平穏な日常の裏にある不穏さ
- 『進撃の巨人』の壁の中 — 閉じた世界と外への渇望
- 『十三機兵防衛圏』— 複数視点で紡がれるSF的な物語構造
- 『ハウルの動く城』— 魔法が自然に存在する生活感

## 5. 相談時の質問リスト

外部 AI サービスに相談する際に使える質問:

1. 上記の設定で、「人間プレイヤーが仮想世界にアクセスする経緯」として、夢の延長という方向性を具体的なストーリーに落とし込んでほしい。ゲーム起動時に表示される短い導入テキスト（300字程度）も提案してほしい。

2. 荘園の背景設定をもう少し掘り下げてほしい。特に「不穏さの正体」と「転送の仕組み」について、複数の案を出してほしい。

3. 上記の世界観とトーンに合うゲームタイトルを10個程度提案してほしい。日本語・英語・造語いずれも可。

4. 人間プレイヤーと荘園の住人の「初めてのコンタクト」シーンを描写してほしい。住人にとって「天の声」が初めて聞こえた瞬間の反応を中心に。

5. 荘園の日常シーン（穏やかだが不穏な日常）の描写例をいくつか書いてほしい。季節や時間帯による変化も含めて。

## 6. 技術的な制約（ストーリーに影響する部分）

ストーリーを考える際に知っておくべき技術的な事実:

- ゲームを閉じると荘園の時間は止まる（キャラクターは「眠る」）
- 人間の介入は「テキストメッセージ」のみ（直接行動はできない）
- AI キャラクターの行動は LLM（大規模言語モデル）が生成する
- キャラクターの記憶は有限（スライディングウィンドウ方式）
- 脱出ワールドは事前に設計されたシナリオに基づく
- 複数のプレイヤーが同時に接続することは当面想定しない
