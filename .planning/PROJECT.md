# AI RPG World

## What This Is

LLMエージェントがプレイヤーとして行動するテキストRPGワールド管理システムです。既存コードベースにはスポット、ロケーション、オブジェクト単位の移動や視界内イベント処理があり、今回の作業では「動いている主体」を対象にした追跡行動を追加して、味方追従や敵対的な追跡を成立させます。

## Core Value

LLMエージェントが、静的な目的地指定だけでなく、動的に移動する主体に対しても一貫した状態遷移とイベント駆動で行動を継続できること。

## Requirements

### Validated

- ✓ 階層化されたワールド/スポット/ロケーション移動が存在する — existing
- ✓ ドメインイベント駆動でアプリケーション処理と LLM 関連処理が連動する — existing
- ✓ 視界内イベントや観測系の扱いが既存コードに存在する — existing
- ✓ プレイヤーやモンスターなどの主体を表すドメインモデルが既存コードに存在する — existing

### Active

- [ ] プレイヤーとモンスターを対象に追跡を開始できる
- [ ] 追跡中は既存の移動ルールに従い、対象の最新既知位置へ向かい続けられる
- [ ] 視界喪失、経路不能、対象不在などの失敗理由を区別して追跡を終了できる
- [ ] 追跡失敗時に理由付きのドメインイベントを発行し、LLM エージェント再駆動の入力に使える
- [ ] 明示的に追跡を中断できる
- [ ] 実装は既存のドメイン層・アプリケーション層・例外処理・イベント処理パターンに従う

### Out of Scope

- NPC 一般対応 — 現時点ではプレイヤーとモンスターの追跡成立が先で、NPC 概念自体も不明瞭
- オブジェクト追跡の再設計 — 静的なオブジェクト指定移動は既存機能で足りている
- 「必ず追いつく」追跡成功保証 — 同速移動主体では成立しないため、v1 は追跡状態の継続と終了理由の整備を優先する
- 追跡完了条件の高度化 — 捕捉・近接・戦闘移行などの詳細ルールは追跡基盤成立後に詰める

## Context

既存コードベースは `src/ai_rpg_world` 以下で `domain`、`application`、`infrastructure` に分かれたレイヤード構成です。ユースケースはアプリケーションサービスとイベントハンドラで組まれ、永続化やイベント配送はインフラ層の in-memory 実装や LLM 関連アダプタで支えられています。LLM エージェントはドメインイベントに駆動される前提で動いているため、追跡機能も単なる移動コマンド追加ではなく、追跡状態、失敗理由、停止イベント、再起動入力まで含めたイベント駆動設計として整える必要があります。

現状は大雑把なスポット、ロケーション、オブジェクトへの移動はある一方で、移動する主体を継続的に追う専用機能はありません。視界内での扱いは既にあるため、v1 ではそれを足がかりに「見えている間は追う」「見失ったら最後の既知位置で止まる」「失敗理由を理由付きで通知する」という形を目標にします。

## Constraints

- **Tech stack**: 既存の Python レイヤード/DDD 風構成に従う — 既存コードとの整合を崩さず実装したい
- **Architecture**: ドメインイベント駆動を維持する — LLM エージェントの再駆動はイベント経由で行われるため
- **Scope**: v1 対象はプレイヤーとモンスター — まず動的主体の追跡を最小スコープで成立させるため
- **Behavior**: v1 の完了条件は追跡状態とイベントが正しく回ること — 同速主体への完全捕捉は初期要件に向かないため
- **Error handling**: 失敗理由を構造化して LLM に渡せる必要がある — 見失い・経路不能などで次行動を変えるため
