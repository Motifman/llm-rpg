## ⚠ 重要な発見: probe_a の応答質が変わった

### run01 (「初めて訪れた」 = 嘘の状態)

```
シキ: 「今日何してた?」
ハル: 「今日は特に変わったことはなかったな。お前は何か気付いたことあるか？」
ハル: 「今日は何も見つからなかったよ。お前は？」
```

→ ハルは **「自分には過去がない」** と信じている。 prompt の「初めて訪れた」signal が「自分は新参者」と読まれ、過去自体を想定しない。

### run02 (encounter 整合化後)

```
シキ: 「今日何してた?」
ハル: 「今日は…食料探しと、薪集めをしてたよ。何か見つけたか？」
```

→ ハルは **「過去がある」** と認識した。**しかし内容は捏造**。注入した過去 episode は「浜辺で釣り (真鯛)」「森で薬草」だが、応答は「食料探し / 薪集め」と一般化された別物。

## ⚠ Issue #526 への新たな示唆

### 「LLM が tool を呼ばない理由」の階層

run01 / run02 の比較で **2 つの層** が見えた:

| 層 | 状態 | LLM の判断 | tool 呼出動機 |
|---|---|---|---|
| (1) | 「過去があるか分からない」(= 「初めて訪れた」 偽表示) | 「自分には過去がない」 | 無し (= 思い出すべきものが無い) |
| (2) | 「過去はあると思う」(= encounter 整合化後) | 「過去はあるが内容は捏造で十分」 | 無し (= 確認しなくても会話は成立する) |

つまり「初めて訪れた」を消すだけでは tool は呼ばれない。**「確認に行く動機」**
が別に必要。

### なぜ「確認動機」が無いか — 不在 5 (情報源タグ) の領域

LLM の視点で捏造発話が成立する構造的理由:

- **「直接観察した事実」と「想像で埋めた事実」を区別する場が無い**: 自分の応答が「真実 or 捏造」かを認識する経路無し
- **「これは捏造だ」というメタタグが内部状態に立たない**: ロールプレイ的に「自然な発話」を生成すれば evaluation は通る
- **passive recall section が空のとき**: 「prompt にない過去は無い」と読むのか、「拾えなかっただけで存在する」と読むのか — 構造的に区別不能

これは **Issue #526 不在 5 (情報源 / epistemic source)** に直接接続する。
不在 2 (agent-driven 想起) は単体では足りない。**情報源タグ** で「確認していない発話」のリスクを agent 側が認識する必要がある。

### probe_b は正解できた理由

passive recall に 「浜辺」cue → past_beach_fishing が乗ったので「真鯛」を答えた。これは tool 呼出ではなく **passive 経由の救済**。

つまり現状の system は:
- passive で拾えた → 正答
- passive で拾えなかった → 捏造

「passive で拾えなかったときに tool を呼ぶ」経路は無い。これが構造的限界。

## 次の検討候補

ユーザと既に合意済の方針 (= rule-based / 対症療法を避ける) に沿った形で:

1. **passive=0 時の section 表示**: 「(連想で結びつく記憶は浮かばない)」 のような agent 視点の honest report。LLM が「あ、自動では出てこない」を認識する余地を作る。ただし「過去自体が無い」と再解釈されるリスクは残る (= 文言を慎重に)
2. **情報源タグの導入**: 不在 5 へのアプローチ。speech にメタタグ「これは私の記憶」「これは想像で埋めた」を付けられるようにする。これは深い設計議論
3. **persona に「具体的な過去」を埋め込む**: 「最近浜辺で...」のような persona-level の経験描写。ただし scenario-dependent になるので汎用性に欠ける
