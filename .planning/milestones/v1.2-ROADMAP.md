# Milestone v1.2: World Simulation Service Refactoring

**Status:** ✅ SHIPPED 2026-03-14
**Phases:** 11-13
**Total Plans:** 6

## Overview

v1.2 では、`WorldSimulationService` の巨大な tick 処理を、既存の順序保証と wiring 契約を壊さずに段階的に分割しました。6-stage facade への整理、monster lifecycle / behavior / hunger migration の責務分離、そしてそれらを継続的に守る simulation regression harness までを 1 つの milestone として完成させています。

## Phases

### Phase 11: Tick Facade Extraction

**Goal**: `WorldSimulationService` が tick 全体の facade として残りつつ、主要な world progression stage を明確な委譲境界で実行できる
**Depends on**: Nothing
**Plans**: 2 plans

Plans:

- [x] 11-01-PLAN.md - facade の前半 stage composition を導入し、environment / movement / harvest を安全に委譲する
- [x] 11-02-PLAN.md - active spot 以降の stage と post-tick wiring を抽出し、6-stage facade を完成させる

**Details:**
`WorldSimulationApplicationService` から environment / movement / harvest / monster lifecycle / monster behavior / hit box の 6 stage を明示 collaborator として切り出し、post-tick の `llm_turn_trigger` / `reflection_runner` を facade 側の責務として固定した。tick 全体の順序と既存 runtime path を読みやすい shape に整理した。

### Phase 12: Monster Policy Separation

**Goal**: monster lifecycle / behavior 周辺の業務ルールが、`WorldSimulationService` 本体から独立した policy / 協調オブジェクトとして読める
**Depends on**: Phase 11
**Plans**: 2 plans

Plans:

- [x] 12-01-PLAN.md - pure hunger migration policy と monster behavior coordinator/rule を先に抽出する
- [x] 12-02-PLAN.md - lifecycle survival 境界と hunger migration apply orchestration を facade 契約維持のまま整理する

**Details:**
`HungerMigrationPolicy`、`MonsterBehaviorCoordinator`、`MonsterLifecycleSurvivalCoordinator` などの協調オブジェクトを導入し、monster lifecycle / behavior の業務ルールを facade 本体から外した。`lifecycle -> behavior -> hitbox` の順序と blocked actor handoff は facade 契約として維持した。

### Phase 13: Simulation Regression Harness

**Goal**: world simulation の責務分割後も、順序保証と主要副作用を継続的に検証できる回帰テスト基盤を持つ
**Depends on**: Phase 12
**Plans**: 2 plans

Plans:

- [x] 13-01-PLAN.md - facade regression を契約別 integration harness に切り分け、builder/helper を薄く整える
- [x] 13-02-PLAN.md - movement / monster lifecycle / monster behavior stage の guard regression を専用テストで追加する

**Details:**
world simulation 回帰を `stage order` / `active spot save` / `post-tick hook` の契約別 integration tests と、movement / lifecycle / behavior の stage-level regressions に分けた。これにより、facade-level の退行と stage-level の退行を別々に検知できるようになった。

## Milestone Summary

**Decimal Phases:**

- None

**Key Decisions:**

- Decision: `WorldSimulationService` は薄い facade として残し、world progression は explicit stage collaborator へ委譲する
- Decision: `llm_turn_trigger` と `reflection_runner` の post-tick hook は facade に残し、stage に分散しない
- Decision: hunger migration / monster behavior / survival progression は policy / coordinator に分割し、monster 業務ルールを facade 本体から外す
- Decision: 回帰テストは巨大な 1 ファイルへ積み増さず、契約別 integration と stage-level regression に分けて失敗診断を明確にする

**Issues Resolved:**

- `WorldSimulationService` に集中していた巨大 tick 処理の可読性と変更容易性の低さ
- monster lifecycle / behavior と hunger migration の業務ルール境界の不明瞭さ
- 分割後の world simulation を継続的に守る regression harness 不足

**Issues Deferred:**

- `MovementService` 自体の詳細分割
- `ToolArgumentResolver` の別ストリーム refactor との統合判断
- `PhysicalMapAggregate` の trigger / interaction / harvest 責務整理

**Technical Debt Incurred:**

- v1.2 は SUMMARY.md / VALIDATION.md / milestone audit / targeted regression runs を主証跡としており、専用の phase-level `VERIFICATION.md` は未作成
- async event failure visibility は現行 UoW/event pipeline で依然として弱い
