# Milestone v1.0: Pursuit Foundation

**Status:** ✅ SHIPPED 2026-03-12
**Phases:** 1-7
**Total Plans:** 19

## Overview

v1.0 では、AI RPG World に動的主体追跡の基盤を導入しました。プレイヤーとモンスターが共有する pursuit 語彙、プレイヤー追跡開始/中断、world tick での継続追跡、observation/LLM 連携、モンスター側整合、live runtime 配線、監査証跡の補完までを 1 つのマイルストーンとして完了しています。

## Phases

### Phase 1: Pursuit Domain Vocabulary

**Goal**: 追跡を静的移動と分離した明示的なドメイン概念として定義する
**Depends on**: Nothing (first phase)
**Plans**: 3 plans

Plans:

- [x] 01-01: 追跡状態と失敗理由のドメイン型を定義する
- [x] 01-02: 追跡ライフサイクルイベントを定義する
- [x] 01-03: プレイヤー/モンスターの既存状態モデルへの組み込み方針を整える

**Details:**
追跡状態、最後の既知位置、構造化失敗理由、追跡開始/更新/失敗/中断イベントを導入し、静的な移動状態と独立した追跡語彙を確立した。

### Phase 2: Player Pursuit Commands

**Goal**: プレイヤーが追跡開始/中断を明示的に行えるようにする
**Depends on**: Phase 1
**Plans**: 2 plans

Plans:

- [x] 02-01: プレイヤー追跡コマンドとバリデーションを追加する
- [x] 02-02: 中断コマンドと追跡状態クリアを実装する

**Details:**
プレイヤーが可視なプレイヤー/モンスターを追跡対象に指定できる application service と LLM tool path を追加し、移動中断と別物としての pursuit cancel を定義した。

### Phase 3: Pursuit Continuation Loop

**Goal**: 最新既知位置に基づく継続追跡を既存移動フローの中で成立させる
**Depends on**: Phase 2
**Plans**: 3 plans

Plans:

- [x] 03-01: tick 時の追跡継続/再計画ロジックを追加する
- [x] 03-02: 最後の既知位置の更新と可視喪失ハンドリングを実装する
- [x] 03-03: 継続追跡の回帰テストを追加する

**Details:**
world tick 中で pursuit continuation を評価し、対象可視時の refresh、視界喪失後の last-known continuation、path_unreachable / target_missing / vision_lost_at_last_known を含む失敗遷移を成立させた。

### Phase 4: Observation And LLM Delivery

**Goal**: 追跡の終了理由と中断結果を observation/LLM パイプラインへ接続する
**Depends on**: Phase 3
**Plans**: 3 plans

Plans:

- [x] 04-01: pursuit イベントの recipient/formatter/registry wiring を追加する
- [x] 04-02: LLM 再駆動に必要な structured payload を整える
- [x] 04-03: interruption と outcome の統合テストを追加する

**Details:**
追跡イベントを既存 observation handler registry に流し、失敗理由や interruption scope を構造化 payload として整え、追跡 outcome から LLM turn resumption をスケジュールできるようにした。

### Phase 5: Monster Pursuit Alignment

**Goal**: モンスター側の追跡表現を新しい追跡基盤と整合させる
**Depends on**: Phase 4
**Plans**: 2 plans

Plans:

- [x] 05-01: モンスター側の追跡状態/理由語彙を整合させる
- [x] 05-02: モンスター追跡回帰テストを追加する

**Details:**
モンスターの `CHASE` / `SEARCH` 遷移で shared pursuit vocabulary を使えるようにし、last-known 維持、再捕捉、探索失敗 cleanup を既存 world simulation seam で受け入れた。

### Phase 6: Player Pursuit Runtime Assembly Closure

**Goal**: プレイヤー追跡の live runtime 配線を完成させ、追跡開始から継続・観測再開までを実行経路上で成立させる
**Depends on**: Phase 5
**Plans**: 3 plans

Plans:

- [x] 06-01: live runtime で pursuit command/continuation services を組み立てる
- [x] 06-02: player pursuit の start -> tick continuation -> observation resumption を接続する
- [x] 06-03: 非テスト bootstrap を通る回帰テストを追加する

**Details:**
`compose_player_pursuit_runtime(...)` を authoritative な non-test runtime seam として追加し、pursuit command service と pursuit continuation service を同じ runtime package に束ねた。

### Phase 7: Pursuit Audit Evidence Backfill

**Goal**: 監査を阻害している verification/validation 成果物と要件トレースの欠落を解消する
**Depends on**: Phase 6
**Plans**: 3 plans

Plans:

- [x] 07-01: Phase 3 と Phase 5 の verification evidence を再構築する
- [x] 07-02: Phase 4 の verification/validation artifacts を完了させる
- [x] 07-03: requirements traceability を監査結果に合わせて同期する

**Details:**
監査に必要な verification/validation と traceability を current codebase に合わせて補完し、v1.0 planning ledger をアーカイブ可能な状態へ戻した。

## Milestone Summary

**Decimal Phases:**

- None

**Key Decisions:**

- Decision: 追跡状態は静的移動状態から分離する
- Decision: `cancelled` は failure reason ではなく独立イベントとして扱う
- Decision: プレイヤー pursuit の authoritative runtime は low-level bootstrap ではなく専用 composition entrypoint で閉じる
- Decision: `OBSV-02` の最終受け入れは observation wiring 単体ではなく live runtime closure に置く

**Issues Resolved:**

- プレイヤー pursuit の non-test runtime 配線不足
- Phase 3/4/5 の verification 欠落
- REQUIREMENTS と verification ownership の不整合

**Issues Deferred:**

- Phase 1 の Nyquist validation artifact 未作成

**Technical Debt Incurred:**

- busy-state timing と observation-driven movement interruption の回帰面は今後も監視が必要
- 現行 UoW/event pipeline は async event failure visibility が弱い
