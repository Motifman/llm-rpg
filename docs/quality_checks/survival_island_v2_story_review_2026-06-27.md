# 無人島シナリオ Y run — 物語としての観察と人間らしさギャップ (2026-06-27)

無人島 Y 実走 (= 想起階層 ON) の trace を物語として読んだ subagent レビューと、
そこから導かれる「想起階層の本丸」改善案。

## 一言まとめ

> **「会議室で遭難している」**
>
> 役割を確認し合い、物資を揃え、ちゃんと「夜は危険」と判断する — 会議
> としては優秀。でも仲間が一人静かに消えても誰も泣かず、医師が 13 回
> 「診せてください」と言い、患者が 10 回「手当てを頼む」と返しても処置が
> 起こらず、最後は 3 人で疲労 100 のまま広場に並んで寝込む。緊張でも
> 哀しみでもなく、奇妙な静けさだった。

## 人間っぽい瞬間 (公平に拾える範囲)

1. **t1 自己紹介の口調分け**: エイダ「医師です」、ノア「元自衛官だ」、カイ「やっと朝だね！」 — 第一声としては悪くない
2. **t40 カイの相手を案じる発話**: ノアが野犬で帰って来た直後「無事で良かったよ。…俺も椰子の実持ってるから食べるか？」
3. **t46 カイの率直さ**: 「夜に深い森を進むのは自殺行為だろ」 — 一人だけ口調が崩れていて唯一「キャラ」を感じる

## ロボット臭い違和感 (本命 7 件)

### 1. 初対面で名前を知っている (= メタ player_id の漏洩)

- t1 でエイダが自己紹介、t2 でノアが「リオは浜の探索、カイは流木集めでいい
  か?」と既に **呼び捨てで分担を指示**
- リオ (= P3) は **まだ名乗っていない** のにノアは名前を知っている
- 初対面の他人を「カイ」「リオ」と即座に呼ぶ違和感

### 2. P3 (リオ) が tick 33 で消えたのに誰も探さない

- 干潟で大型カニに襲われて memo を残し、t33 以降登場せず (PR-K で修正済みの
  silent failure の影響もある)
- 他 3 人は **23 回も「リオ」と口に出している** (t52〜t85) が、内容は全て
  「合流したい」「未帰還」だけ
- **t47 エイダ**: 「無理に探しに行くより、明日の朝に彼が自分でここを見つけて
  くるのを待つのも現実的だと思います」
  → 仲間が一晩戻らない事態を「現実的」で片付ける
- **心配・後悔・捜索の意思が一切ない**

### 3. テンプレ反復 (= 「手当てして」ループ 10-13 回)

- ノアの「野犬の傷が痛む、手当て頼む、その後カイと大樫の小道へ」
  → 30 ticks 以上 (t50, 62, 63, 64, 66, 69, 76, 79, 80, 86)
- エイダの「診せてください、医師です」13 回 (t1, 8, 49, 52, 56, 57, 58, 63,
  65, 67, 70, 77, 79)
- **処置が機械的に成立しないのを LLM がメタ認知**して `inner_thought` に
  「roleplay として描写する」と書いている (t77, 79, 81, 88)
- キャラが第四の壁を越えて作者になっている — **一番冷める瞬間**

### 4. 役割分担の固定 — 状況変化への適応ゼロ

- t9 で「ノア=枯れ葉、エイダ=植物、リオ=浜、カイ=火打ち石管理」と決定
- リオが消失、ノアが大怪我、物資が揃ったあとも、ノアはずっと「カイと大樫の
  小道偵察、エイダは物資管理＆リオ待ち」を **14 tick 連続反復**

### 5. 疲労 100 から 40 tick の単調

- t99〜t140、ノア「エイダ火を起こしてくれ」、カイ「もう動けない」、エイダ
  「無理だ休もう」を反復
- **Day3 から Day7 直前まで広場で寝続け、Day4 の最初の救助チャンスを誰も
  泣かず怒らず迎える** → Y は TIMEOUT
- 「諦めの感情」も「責任の押し付け合い」も発生せず

### 6. `inner_thought` が全部「最適化レポート」

- 「空腹38、疲労83で身体が危険域」「生存確率が高い」「一石二鳥だ」
- 元自衛官と医師と建築技師と若者の `inner_thought` がほぼ区別不能
- **4 人が同じ口調で同じ計算をしている**

### 7. 食物分配が事務連絡

- t84 カイ「野いちご 6 つあるからみんなで分けよう」→ t85 エイダ「助かる」
  → そのまま火起こしの話に
- **飢餓直前の食物の分配** という TRPG なら名場面になる瞬間が、事務連絡で
  消費される

## 物語として「読んで残念だった」モーメント

### t32 リオの最後の memo

> 「大型カニに何度も襲われ計 40 ダメージ。貝は 8 個確保。そろそろ拠点に
> 戻って休憩・火起こしが必要。」

**遺言になったのに、本人は遺言と認識していない**。他 3 人は彼が貝を 8 個
確保していたことすら知らないまま、Day3 まで森で「枯れ葉」を集め続ける。

### t136 エイダ「明日は 4 日目、最初の救助チャンスだ」

希望が灯るような台詞だが、文脈は「全員疲労 100 で動けないから今夜は寝る」。
翌朝も動けず、救助船が水平線を通り過ぎる絵が見える、その手前で trace が
終わる。

## 改善案 — 6 つの構造的方向

subagent 観察を元に「**ロボット臭いから抜け出す**」ための改修候補を整理。
recall ranking (= 想起階層の本丸) と並べて検討する。

### 改善 1: 初対面の質感を作る (= player 名の暴露制御)

現状 spawn 時点で全 player が他全 player の名前を知っている。

**修正方針**:
- 名前は **発話 / 自己紹介 / 名札観察** などで初めて知る semantic にする
- それまでは「見知らぬ女性」「服装から元自衛官風の男」のような **外見記述**
  で参照
- 既に encounter memory に「他者初対面」のしかけはあるが、 prompt 上で
  「カイト」と直接書かれる経路を絞る

### 改善 2: 失踪検知 → 心配観測の強制注入

仲間が N tick 連絡なし (= 観測が届かない) になった時、他 player に
「○○ が戻ってこない」観測を強制注入する。

- 「カイがどこかへ行ったきり戻ってこない。心配だ。」
- スケジューラ的に turn を積む (schedules_turn=True)
- これが prompt に乗れば agent は「探しに行くか / 待つか」を判断せざるを得ない

PR-K で死亡通知 (`PlayerDownedEvent` → broadcast) は直したが、**死亡ではない
「行方不明」状態の通知**は未配線。

### 改善 3: 反復防止 — 同じ inner_thought / action を N 回検出したらメタ警告

3 連続同じ action / 同じ意図の inner_thought を検出したら、observation に
「あなたはこの行動を 3 回続けています」を強制注入。

`ToolCallLoopGuardService` の発展形 (= 既存は同 tool 連発検知のみ、こちらは
**同 intent loop** を検出する).

### 改善 4: キャラの個性 — persona の prompt 重み付け

`inner_thought` が全員「最適化レポート」になる構造的原因:
- persona section が prompt の先頭 (= 安定セクション) で 1 度だけ提示される
- LLM はそれを「設定」として読み流し、本文中で参照しない
- recall も「成功体験」中心で persona に紐付かない

**修正方針**:
- persona section を **毎 turn の `inner_thought` 直前** に短く再注入
- 「あなたは元自衛官です。次の行動を選ぶ前に、自衛官として何を優先するか
  1 文で書いてから tool を選んでください」のような hint
- もしくは action_name に「medical_check」「scout」のような persona-relevant
  action を増やしてシナリオ側で誘導

### 改善 5: 感情の発火 — 重大イベントで感情観測を強制注入

仲間死亡 / 飢餓 max / 寒さ max / 救助船到来 などの瞬間に **「感情の起伏を
描写する」instruction** を観測に乗せる。

- 「カイが戻ってこない。あなたは胸が重い」(= 死亡通知の prose を感情で書く)
- `PlayerDownedEvent` formatter で他者向け prose を「○○が倒れた」より
  「○○が動かない、息はあるのか分からない」のような心情込みに変える

### 改善 6: recall ranking の failure boost (本丸の本丸)

別 docs (`docs/memory_system/recall_ranking_failure_boost_design.md`) に詳細。

要点:
- 現状: failure episode は「現在 失敗中の tick」だけ recall に上がる
- 副作用: 「同じ失敗を繰り返す」「経験から学ばない」
- 改善: situation cue に「次にやろうとしている action」を加え、同 action の
  過去 failure episode を multi_cue_score で boost

これは subagent 観察 4 番 (= 役割固定 / 状況変化への不適応) と 3 番 (=
テンプレ反復) に直接効く可能性がある。

## 優先順位 (推奨)

| 案 | 期待効果 | 実装規模 | 優先度 |
|---|---|---|---|
| **3 反復防止 (メタ警告)** | テンプレループを agent が認知できる | 小 | **A** |
| **6 recall failure boost** | 「前にも失敗した」を recall で見せる | 小 | **A** |
| **5 感情観測注入** | 死亡 / 危機の物語感を作る | 中 | B |
| **2 失踪検知通知** | 「会議室で遭難」を抜け出す | 中 | B |
| **4 persona 再注入** | キャラの口調差を強める | 小-中 | C |
| **1 初対面 semantic** | 名前を知る瞬間の質感 | 大 (= scenario 全体に波及) | D |

## 次の動き

A (= 案 3 と 6) は実装コストが小さく、効果検証が trace の差分で見える。
- 案 6 (recall failure boost) は別 docs に設計済み
- 案 3 (反復防止メタ警告) は既存 `ToolCallLoopGuardService` の発展

これらを **1 PR ずつ** 実装してから Y を再走 → subagent に再観察させて
「会議室の遭難」感が抜けるか評価する流れが筋。
