# Milestone v1.1: LLM Skill Tooling

**Status:** ✅ SHIPPED 2026-03-13
**Phases:** 8-10
**Total Plans:** 10

## Overview

v1.1 では、LLM が既存 runtime 文脈の中でスキル装備、進化提案の受諾/却下、覚醒モード発動を安全に扱えるようにしました。typed runtime target、label-driven tool contract、executor/wiring の実行経路、awakened の hidden-first availability、observation/runtime proof、UAT と milestone audit までを 1 つのマイルストーンとして完了しています。

## Phases

### Phase 8: Skill Runtime Context And Tool Contracts

**Goal**: skill 系ツールを LLM に安全に見せるための runtime target・tool definition・argument resolution を整える
**Depends on**: v1.0 runtime and observation foundation
**Plans**: 4 plans

Plans:

- [x] 08-00: Nyquist Wave 0 として不足しているテストアンカーを追加する
- [x] 08-01: skill runtime read models と typed runtime target DTO を整える
- [x] 08-02: UI label 表示と label-to-canonical argument resolution を追加する
- [x] 08-03: public tool names / schemas / availability / available-tools coverage を閉じる

**Details:**
current-state から equip / proposal / awakened 用の read model と typed runtime target を構築し、`EK` / `ES` / `SP` / `AW` ラベルを prompt と resolver に接続した。`skill_equip`、`skill_accept_proposal`、`skill_reject_proposal`、`skill_activate_awakened_mode` の公開契約と availability を current-state label presence に揃えて固定した。

### Phase 9: Skill Equip And Proposal Decision Tools

**Goal**: LLM が装備変更と進化提案の受諾/却下を行えるようにする
**Depends on**: Phase 8
**Plans**: 3 plans

Plans:

- [x] 09-01: skill equip / proposal decision の facade と world executor 実行経路を接続する
- [x] 09-02: proposal target の slot-aware display 情報と Phase 9 成功文面を確定する
- [x] 09-03: mapper / wiring / runtime path 回帰で Phase 9 ツール群の統合を証明する

**Details:**
`skill_equip`、`skill_accept_proposal`、`skill_reject_proposal` を既存 skill facade / world executor / mapper 経路に載せ、表示用 slot 情報を typed runtime target に通したまま結果文面へ反映した。default LLM wiring 上でも `state -> ui labels -> resolver -> mapper execute` が通ることを回帰テストと UAT で確認した。

### Phase 10: Awakened Mode Tooling And Runtime Proof

**Goal**: 覚醒モード発動を LLM tool として公開し、runtime path で安全性と観測結果を確認する
**Depends on**: Phase 9
**Plans**: 3 plans

Plans:

- [x] 10-01: awakened activation の facade defaults と world executor 実行経路を接続する
- [x] 10-02: hidden-first な awakened visibility を resource 条件まで拡張する
- [x] 10-03: default wiring と runtime path 回帰で awakened tool の統合を証明する

**Details:**
覚醒モード発動を server-side defaults 前提で skill facade に追加し、内部数値を LLM に渡さず短い成功結果を返す executor seam を実装した。さらに resource sufficiency を含む hidden-first gating を current-state に取り込み、observation/runtime confirmation まで含めた awakened path の共存を回帰テストと UAT で閉じた。

## Milestone Summary

**Decimal Phases:**

- None

**Key Decisions:**

- Decision: skill 系ツールは人間向けラベルから canonical id に解決し、raw id を LLM 契約へ出さない
- Decision: availability は current-state label presence に合わせた hidden-first を採用する
- Decision: 覚醒モードは LLM が発動判断のみを行い、コストや duration は server-side defaults で固定する
- Decision: runtime proof は isolated unit test だけでなく default wiring と observation path まで含めて証明する

**Issues Resolved:**

- skill equip / proposal / awakened を既存 LLM tool family に安全に追加するための label/runtime contract 不足
- equip / proposal の executor wiring と user-facing success semantics の未確定状態
- awakened activation の server-side defaults、visibility gating、runtime/observation proof の欠落

**Issues Deferred:**

- pursuit の follow/chase 詳細化
- group / formation control
- 覚醒モードの効果詳細や数値表現の richer user-facing 化

**Technical Debt Incurred:**

- v1.1 は dedicated phase-level `VERIFICATION.md` を残しておらず、archive 判断は SUMMARY / VALIDATION / UAT / milestone audit に依存している
- async event failure visibility は現行 UoW/event pipeline で依然として弱い
