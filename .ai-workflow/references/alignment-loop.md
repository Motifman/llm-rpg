# Alignment Loop

`flow-*` の中心目的は、ユーザーが思い描くものと、エージェントが内部で想定しているもののズレを小さくすることにある。

そのため、`idea`、`plan`、`exec` にはそれぞれ短い alignment loop を入れる。

## 共通原則

- いきなり完成版を出して終わらない
- 暫定解釈を一度提示し、ズレがあるかを確認する
- コード調査で見えたコストや制約は、質問の材料として使う
- 質問は「どの点でズレやすいか」に集中させる
- ループ結果は artifact に残す

## `flow-idea`

目的:
- 何を作りたいのか
- 何をもって成功とするのか
- 今回やらないことは何か

最低 1 回は alignment loop を回す。

推奨ループ:
1. アイデアの暫定解釈を一文でまとめる
2. 関連コードを軽く調査する
3. 「この理解で合っているか」「成功条件はどこか」「非目標は何か」を聞く
4. 回答を artifact に反映する

## `flow-plan`

目的:
- phase 分けと順序の妥当性
- 実装コストに対する期待値調整
- 成功条件の明確化

最低 1 回は draft plan に対する alignment loop を回す。

推奨ループ:
1. コード調査をもとに暫定 phase 案を作る
2. 「この順序でよいか」「重い部分を含めるか」「何をもって完了とするか」を聞く
3. 反応を受けて plan を修正する
4. 修正版を artifact に残す

## `flow-exec`

目的:
- 実装中に更新された内部モデルと plan のズレを埋める
- 次 phase への引き継ぎを明示する

各 phase 完了時に 1 回は feedback loop を回す。

推奨ループ:
1. 実装中に得た観測情報を列挙する
2. 元の phase 想定と何が違ったかを見る
3. その差分が次 phase にどう影響するか整理する
4. 全体目的に照らして plan と progress を更新する
